# Unity Editor 実機確認チェックリスト

Steam向け短編2D SRPG — Unity Editor上での実機確認手順。
各項目は「☐」から「✅」へ書き換える。

---

## 1. 事前準備

- [ ] Unity 6000.4.9f1 またはプロジェクトで使用中のUnityバージョンがインストールされている
- [ ] プロジェクトを正常に開けることを確認
- [ ] コンソールに既存のエラーが累積していないことを確認
- [ ] 電源供給・モニター環境が安定している
- [ ] 確認結果記録欄（項19）を用意しておく

---

## 2. Unity Editor 起動確認

- [ ] Unity Editor が正常に立ち上がる
- [ ] コンソールウィンドウに新規エラー・予期せぬWarning がない
- [ ] 初期画面としてタイトル画面、またはタイトル相当のUIが表示される
- [ ] Game ビューの解像度がデフォルトで設定されている（1920×1080推奨）

---

## 3. Console エラー確認

- [ ] Player Log に Error が存在しない
- [ ] Console に新しい Error / Exception が追加されていない。Warning は内容を確認し、既知または意図したものか判断する
- [ ] Script の Missing Reference / NullReference が解消済み
- [ ] AssetBundle / Resources に未解決参照がない

---

## 4. タイトル画面確認

- [ ] ゲームタイトルが表示されている
- [ ] 「New Game」「Options」「Quit」のメニュー項目が存在する
- [ ] **Up / Down Arrow** で項目が選択・移動できる
- [ ] **Enter** で各項目を決定できる
- [ ] New Game 選択 → ステージ選択画面へ遷移
- [ ] Options 選択 → オプション画面へ遷移
- [ ] Quit 選択 → Editor が停止（または再生終了）する
- [ ] 背景BGMが再生されている

---

## 5. Options 画面確認

- [ ] Master Volume / BGM Volume / SE Volume のスライダーが表示される
- [ ] **Up / Down Arrow** で項目の移動、**Enter** で決定・切替が可能
- [ ] Mute ON/OFF が機能する
- [ ] Master Volume 0% にすると全音声が静かに聞こえなくなる
- [ ] BGM Volume 調整でBGMのみ減る
- [ ] SE Volume 調整でSEのみ減る
- [ ] **Left / Right** で音量値が微調節できる
- [ ] **Escape** でタイトル画面へ戻る
- [ ] Editor再起動後に設定が保持されていることを確認

---

## 6. ステージ選択画面確認

- [ ] 全6ステージが一覧表示されている
- [ ] **Up / Down Arrow** でステージを選択できる
- [ ] **Enter** で選択したステージを開始できる
- [ ] **数字キー 1〜6** で直接該当ステージを開始できる
- [ ] **Escape / Backspace** でタイトル画面へ戻る

---

## 7. Stage 1 開始確認

- [ ] ステージ1が読み込まれる（チュートリアル面）
- [ ] グリッドマップが表示され、ユニットが配置されている
- [ ] ユニットのHPバー・アイコンが正常に表示される
- [ ] ターン表示が「Player Turn: 1」または同等で始まる
- [ ] 戦闘BGMがタイトルBGMから切り替わる

---

## 8. 基本操作確認

> マウス＋キーボードの操作を以下の順序で試す。

- [ ] **味方ユニットをクリック** → ユニットが選択されハイライトされる
- [ ] **空タイルをクリック** → 移動範囲内であればユニットが移動する
- [ ] **敵ユニットをクリック** → 射程内であれば攻撃が実行される
- [ ] 選択していない状態で敵・障害物をクリック→反応なし（正常）

---

## 9. 移動 / 攻撃 / 待機 / Undo 確認

戦闘中で味方ユニットを選択した状態から以下を確認：

- [ ] **W** → 選択中の味方ユニットが待機し、ターンが遷移する
- [ ] 移動後・行動確定前まで **U** → 位置が元に戻される（Undo）
- [ ] 攻撃後や待機後には Undo が効かない（正常動作）
- [ ] 敵ターンへ移行する前に全ユニットを行動可能/待機させられる

---

## 10. 敵ターン確認

- [ ] 味方すべての行動終了後に敵ターンへ自動的に移行する
- [ ] 敵が移動範囲内に来れば攻撃を行う
- [ ] 敵の距離計算・優先度が期待通りである
- [ ] 敵によるユニット撃破が正常に処理される
- [ ] コンソールに敵AI関連の例外がない

---

## 11. 勝利 / 敗北確認

### 勝利条件
- [ ] 全敵撃破 → バトルログに「VICTORY」表示
- [ ] **Enter** → 次ステージへ自動遷移（ステージ6以外の場合）
- [ ] ステージ6クリア時は「ALL CLEAR」表示がされる

### 敗北条件
- [ ] 自軍全滅 → バトルログに「DEFEAT」表示
- [ ] 画面が一時的に停止/演出された後、リトライ待ち状態になる
- [ ] Console に予期せぬエラーが出ていない

---

## 12. リトライ / 次ステージ確認

- [ ] リトライ時：**R** または **Enter（敗北画面）** で同じステージが再開始される
- [ ] 次ステージ時：**Enter** 押し → ステージデータが切り替わる
- [ ] Stage Transition のフェード/トランジションが表示される
- [ ] **Escape** または **S** → ステージ選択画面へ戻る

---

## 13. 音量 / BGM / SE 確認

> Options 画面で調整しながら以下の項目を実機で確認する。

- [ ] Master Volume を最大/最小/中間 で切り替えた時の音量変化を聴く
- [ ] BGM Volume 0% の時はBGMのみ消え、SEが鳴ったまま
- [ ] SE Volume 0% の時は攻撃音などが消える
- [ ] Mute をONにすると全音声がミュートされる
- [ ] 各ステージで異なるBGM（または同一でも）が再生される
- [ ] 複数音源の同時再生時に破音・割れがない

---

## 14. 解像度確認

- [ ] Gameビューで 1920×1080（Full HD）での描画を確認
- [ ] Gameビューで 1280×720（HD）での描画を確認
- [ ] UI が両解像度で崩れていないことを確認
- [ ] ピクセルアートの拡大率が適切に処理されている
- [ ] カメラのOrtho Sizeが正しく設定されている

---

## 15. Debug.Log / DevLogger 確認

> `DebugLogAudit.md` と併用する。

- [ ] Editor上では `DevLogger.Log` がConsoleに出力される（UNITY_EDITOR有効時）
- [ ] Release build では開発用ログが抑制される方針であることを確認
- [ ] Debug.LogWarning / Debug.LogError は原則として残す方針である
- [ ] Victory / Defeat / ALL CLEAR / 評価ログは未置換で残されていることを確認
- [ ] PlayerController の選択・ハイライト・移動範囲ログが開発用ログへ置き換え済みであることを確認
- [ ] TurnManager の敵AI行動ログが条件付きになっていることを確認
- [ ] StageManager の画面遷移ログが開発用ログへ置き換え済みであることを確認

---

## 16. 6ステージ通し確認

> ステージ1から段階的にクリアしながら動作を確認する。

- [ ] ステージ1（チュートリアル）で初期行動・移動・攻撃を試す
- [ ] ステージ2〜5 を順にプレイし、敵パターン・地形が期待通りであることを確認
- [ ] 各ステージ間でフェード/トランジションが動作することを確認
- [ ] ステージ6（最終面）で「ALL CLEAR」が表示されることを確認
- [ ] 全ステージクリア後にタイトル画面へ帰還できることを確認

---

## 17. Windows Build 前の最終確認

- [ ] Unity Player Settings が正しく設定されている（Company / Product / Version）
- [ ] Scripting Backend が IL2CPP または Mono で適切に設定されている
- [ ] Api Compatibility Level が .NET Standard 2.0 / 4.x に合致している
- [ ] Build Setting でシーン順序が正しい（Title → Stage1〜6、または動的読み込み）
- [ ] DEVELOPMENT_BUILD の OFF/ON を使い分けていることを確認
  - ON: Editor検証・ログ出力用
  - OFF: Steamリリースビルド用

---

## 18. 既知の注意点

- DevLogger.cs は UNITY_EDITOR または DEVELOPMENT_BUILD の条件付きで動作する。
- Release build（UNITY_EDITOR / DEVELOPMENT_BUILD が無効）では開発用 `DevLogger.Log` が抑制される。
- Debug.LogWarning / Debug.LogError は原則抑制されず、リリースでも出力される。
- Victory / Defeat / ALL CLEAR / 評価ログは未置換で残しているため、必要に応じてConsole上で確認する。
- Undo（Uキー）は移動後・行動確定前まで有効である。確定後には無効となる。
- 敵ターン中にユーザー入力を抑止する仕組みが TurnManager に存在する。
